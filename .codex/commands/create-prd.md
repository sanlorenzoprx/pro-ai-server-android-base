# Create PRD

Create a product requirement document before planning significant features.

Rules:

- Clarify the user problem.
- Define target behavior and non-goals.
- Identify affected users and workflows.
- Include acceptance criteria.
- Include validation expectations.
- Do not implement code.

Output:

```text
.agents/PRDs/{feature}.prd.md
```

PRD format:

```md
# PRD: {Feature}

## Problem

## Goals

## Non-Goals

## User Stories

## Requirements

## Acceptance Criteria

## Risks

## Validation
```
